# Programming paradigms
Programming paradigms are different styles or approaches to writing and organizing code, each based on a particular set of concepts and principles that guide how programs are structured and how problems are solved.

## Types 

### 1. Imperative Paradigms

#### i. Procedural Programming
Idea: Program as a sequence of step by step instructions  
Key features: functions or procedures, top down approach, explicit control flow, use of variables  
Use case: system programming, embedded systems, simple scripts

#### ii. Object Oriented Programming
Idea: Model real world entities as objects with data and behavior  
Key features: classes and objects, encapsulation, inheritance, polymorphism, abstraction  
Use case: large scale software, enterprise applications, game development

#### iii. Structured Programming
Idea: Improve procedural programming by enforcing clear structure  
Key features: no goto statements, use of loops and conditionals, modular design  
Use case: writing clean and maintainable procedural code

### 2. Declarative Paradigms

#### i. Functional Programming
Idea: Use pure functions and avoid changing state  
Key features: immutability, higher order functions, no side effects, recursion  
Use case: data processing, parallel computing, artificial intelligence

#### ii. Logic Programming
Idea: Define facts and rules, system derives conclusions  
Key features: rule based logic, backtracking, declarative nature  
Use case: expert systems, knowledge bases, artificial intelligence

#### iii. Query Programming
Idea: Specify what data is required instead of how to get it  
Key features: declarative queries, set based operations, abstraction from implementation  
Use case: databases and data retrieval systems

### 3. Concurrent and Reactive Paradigms

#### i. Event Driven Programming
Idea: Flow controlled by events such as clicks messages or signals  
Key features: event handlers, asynchronous execution, callbacks  
Use case: web applications, graphical user interfaces, real time systems

#### ii. Concurrent Programming
Idea: Multiple tasks execute in overlapping time periods  
Key features: threads or processes, synchronization, shared resources  
Use case: multitasking applications, high performance systems

#### iii. Parallel Programming
Idea: Multiple computations executed simultaneously  
Key features: multi core processing, task parallelism, data parallelism  
Use case: scientific computing, simulations, big data processing

#### iv. Reactive Programming
Idea: Systems react to data streams and changes  
Key features: asynchronous data streams, non blocking execution, event propagation  
Use case: real time applications, streaming systems, responsive user interfaces

### 4. Other Paradigms

#### i. Aspect Oriented Programming
Idea: Separate cross cutting concerns like logging and security  
Key features: aspects, advice, weaving  
Use case: logging, security, transaction management

#### ii. Component Based Programming
Idea: Build systems using reusable components  
Key features: encapsulated components, reusability, modularity  
Use case: user interface frameworks, enterprise systems

#### iii. Dataflow Programming
Idea: Program represented as flow of data between operations  
Key features: nodes and edges, data driven execution  
Use case: signal processing, data pipelines

#### iv. Constraint Programming
Idea: Define constraints and system finds solutions  
Key features: constraint satisfaction, declarative rules, backtracking  
Use case: scheduling, optimization problems

#### v. Scripting Paradigm
Idea: Write lightweight programs for automation  
Key features: interpreted execution, rapid development, high level operations  
Use case: automation, web scripting, system administration

#### vi. Generic Programming
Idea: Write code that works for multiple data types  
Key features: templates or generics, code reuse, type abstraction  
Use case: libraries and reusable components

#### vii. Metaprogramming
Idea: Programs that manipulate or generate other programs  
Key features: reflection, macros, code generation  
Use case: compilers, frameworks, advanced libraries
